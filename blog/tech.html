Ruby's on...what?

A first glance at Rails

March 10, 2016

What is Rails?

Created by David Hansson in 2004, Rails is an opinionated web application development framework, written in Ruby. Ruby as a standalone language is not designed for use on the Web. Enter Rails. Rails provides all the fancy scaffolding and materials needed to write a Ruby application and have it be a website.

Rails streamlines the process of making web applications by making assumptions about what developers need to get started with their app. While there might be multiple ways to accomplish a goal in Rails, the software assumes that there is a "best" way, aka "the Rails way."

What are the guiding principles of Rails?

  • Don't Repeat Yourself (DRY) - means that information is located in a single, unambiguous place.

  • Convention over Configuration (CoC) - which means that a developer only needs to specify unconventional aspects of the application, leading to less code and less repitition.

    Why learn Ruby on Rails?

    It's elegant and awesome. Designed with the best practices in mind, Rails guides you into writing awesome code. Rails is also great for rapid prototyping, which makes it popular amongst tech startups.

    Rails is great for beginners, because Ruby is great for beginners. It's a powerful tool that does most of the hardwork for you.
