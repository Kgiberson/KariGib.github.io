Intro to Arrays and Hashes

What are these Ruby containers and when should I use them?

February 14, 2016

Ruby is an object-oriented programming language, which means that it processes its data as "objects." It is the programmer's job to define the content of the object and how it behaves using Ruby syntax. Ruby represents collections of objects by putting them inside container objects and the two most commonly used containers are...you guessed it!

Arrays and hashes.

Arrays

An array is a collection of objects that is ordered by a numerical index. An array can store any type of object, including other arrays! Each slot in an array is numbered, starting at 0, and it is this numbering that allows a programmer to call a certain object in an array.

array_name = ["objects", "in", "the", "array"]

favorite_foods = ["pizza", "chocolate", "avocado"]

Hashes

Hashes are similar to dictionaries in how they store objects in pairs. Each pair has a key and a value. Any object can serve as a hash key and/or value but keys are unique in each hash (you can't have two keys called "potato" within the same hash.) You must use the key to return the value paired to it.

hash_name = {key1 => "value1", key2 => "value2", key3 => "value3"}

meal_plan = {"Tuesday" => "pizza", "Wednesday" => "tacos", "Thursday" => "salad"}

What's the difference?

Both arrays and hashes are incredibly powerful tools for storing and retrieving data. You can store arrays and hashes inside arrays, and hashes inside hashes. Hashes are faster than arrays at data retrieval. Their major differences as containers lie in how they lookup data.

Arrays use the index (order) number

num_array = [35,-10 ,104]

num_array[1]

(returns -10)

Hashes use the key value:

my_hash = {firstname: "Kari", lastname: "Giberson", hometown: "Staten Island, NY"}

my_hash[:hometown]

(returns "Staten Island, NY")
